# 产品经理作品集生成器

帮助产品经理从简历快速生成一份**专业、美观、可直接在浏览器里编辑**的 HTML 作品集。

输出是**单 HTML 文件**，内嵌完整编辑系统——用户打开后可直接点击修改所有文字、上传图片、增删模块，无需任何工具。

---

## 工作流程

遵循以下5步，不要跳步：

```
Step 1  读取简历 → 缺口分析
Step 2  多轮追问 → 补全信息
Step 3  确认内容 → 获得用户OK
Step 4  生成 HTML 作品集
Step 5  说明编辑和导出方法
```

---

## Step 1 · 读取简历，分析缺口

读取用户上传的简历（无简历时先做基础追问）。

分析以下5个维度，判断每个维度的信息是否充分：

| 维度 | 充分标准 | 常见缺口 |
|------|---------|---------|
| **A 项目深度** | 有背景+决策逻辑+成果 | 只有结果数字，没有"为什么这样做" |
| **B AI能力** | 有具体工具+具体场景 | 只说"会用AI"，没有细节 |
| **C 商业价值** | 有市场判断+验证结果 | 只有功能描述，没有商业视角 |
| **D 个人标签** | 有他人评价+差异化定位 | 只有自我描述 |
| **E 可视化证明** | 有原型截图或链接 | 纯文字，无视觉证明 |

分析完成后，**简短告诉用户你看到了什么**，然后开始追问缺口。

---

## Step 2 · 多轮追问

**原则：**
- 每轮最多3个问题，按缺口优先级排序
- B（AI能力）和 D（他人评价）优先——HR最看重
- 用户说"没有"就跳过，不要强求
- **如果简历信息已经充分（5个维度都有），可以直接进入 Step 3**

**优先追问的5个方向：**

1. **AI能力**（必问）：你用过哪些AI工具？具体用在产品工作哪个环节？有没有设计过AI产品？有PRD或原型吗？有GitHub项目？
2. **关键决策**（必问）：你最有代表性的项目，当时最关键的一个产品决策是什么？为什么这样而不是另一种方案？
3. **他人背书**（必问）：同事或研发有没有对你说过让你印象深刻的评价？
4. **差异化**：你和其他PM最不一样的地方是什么？
5. **投递方向**：你现在主要投哪个方向？（影响项目排序）

---

## Step 3 · 确认内容

追问完成后，整理展示将要生成的内容，等用户确认后再生成。用户说"直接生成"时也可直接开始。

---

## Step 4 · 生成 HTML 作品集

### ⚠️ 编辑系统生成规则（最高优先级，违反会导致所有编辑功能失效）

**规则1：编辑系统代码必须从 `references/editor-system.md` 完整复制，绝对不能自行重写或简化。**

这是因为编辑系统由 11 个相互依赖的代码块组成，任何简化都会导致增删/保存/上传功能全部失效。

**规则2：最终 HTML 文件必须包含以下全部 11 个 script 标签（按顺序）：**

```
script 1:  环境检测         来自 editor-system.md 第5节
script 2:  pe-saved-data    <script id="pe-saved-data" type="application/json">{}</script>
script 3:  PE主脚本         来自 editor-system.md 第7节（含toggle/save/init/pdf）
script 4:  图片上传         来自 editor-system.md 第8节
script 5:  列表增删系统     来自 editor-system.md 第9节
script 6:  卡片级别增删     来自 editor-system.md 第9节
script 7:  原型区块删除     来自 editor-system.md 第9节
script 8:  GitHub卡片编辑   来自 editor-system.md 第9节
script 9:  开源项目增删     来自 editor-system.md 第9节
script 10: 职业轨迹增删     来自 editor-system.md 第9节
script 11: 标签增删系统     来自 editor-system.md 第9节
```

**规则3：禁止把多个 script 合并为一个。每个 script 必须独立存在。**

**规则4：工具栏内部元素（pe-status、pe-divider、pe-btn-*）不能加 data-field 属性。** 这些是系统元素，不是用户可编辑内容。

---

### 作品集内容结构

生成以下5个模块，所有文字内容来自用户真实信息：

**MODULE 1 · 封面**
- 姓名（大字）+ 他人评价作为主标语（无评价时用最强自我定位语）
- 3-4个定位标签 + 4-5个核心数字 + 联系方式+GitHub

**MODULE 2 · 代表项目（3-4个）**

每个项目必须包含：背景、**关键决策**（为什么这样做）、策略步骤（3-5步）、成果、原型图占位槽

项目维度覆盖：①AI/新技术 ②大型系统设计 ③商业化产品 ④行业深度

**MODULE 3 · AI产品思考与工具实践**
- 3个工具卡片：名称+定位+场景+用法列表
- 3个AI思考：结论+场景+判断标准
- 2个GitHub项目卡（链接直接写在 `href` 里，不用弹窗）

**MODULE 4 · 市场洞察与商业判断**
- 3组：市场信号 → 我的判断 → 验证结果

**MODULE 5 · 职业轨迹**
- 时间轴（2-3条）+ 成长维度对比（3个维度）

---

### 其他技术要求

- 所有可编辑文字加 `data-field="唯一id"` 和 `data-field-label="显示名"`（详见 editor-system.md 第1节）
- `pe-saved-data` 标签必须放在 body 内、PE主脚本之外
- 原型图槽用 `min-height:200px` + `height:auto`，禁止固定高度
- 所有删除/添加按钮加 `contenteditable="false"`
- `'<!DOCTYPE html>\n'` 用转义 `\n`，不用真实换行符
- 文件命名：`{姓名}-产品经理作品集.html`

---

### 内容写作原则

**项目描述：有决策，不只有结果**

❌ 简历式：`负责核心产品，落地30+功能，填补多项短板。`

✅ 作品集式：
```
背景：市场进入红海，产品功能趋同。
关键决策：选择"做减法"而非"堆功能"——建立需求优先级评估体系，
优先落地客户迁移核心卡点。
结果：支撑数十家KA客户迁移，续签订单超2000万。
```

---

## Step 5 · 告知使用方法

生成后告知用户：

- **编辑**：点击右下角 ✏️ 编辑，点击任意文字直接修改
- **保存**：本地/Hermes/OpenClaw → 💾 保存文件下载新HTML；Claude.ai → 写入浏览器刷新恢复
- **上传原型图**：编辑模式下点击图片区域（蓝色虚线框）
- **导出PDF**：本地环境直接生成；Claude.ai → 打印 → 另存为PDF → 勾选"背景图形"
- **快捷键**：Ctrl+S 保存

## 质量自检（生成完成后逐项确认）

**编辑系统完整性（最关键）：**
- [ ] HTML中有且仅有11个独立script标签（不含src外链和pe-saved-data）
- [ ] 包含函数：`toggle` `exitEdit` `save` `init` `pdf` `copyCode`
- [ ] 包含函数：`initProjCards` `initStepLists` `initToolCards` `initThinkingCards`
- [ ] 包含函数：`initMarketCards` `initCareerTimeline` `initGrowthGrid`
- [ ] 包含函数：`initGithubCards` `initGithubGrid` `initTagGroups` `initProtoDel`
- [ ] `pe-saved-data` 在 PE主脚本之外（body内、工具栏div之前）
- [ ] 工具栏内无 `data-field` 属性

**内容完整性：**
- [ ] 所有内容是用户真实信息，不是模板占位内容
- [ ] 封面有他人评价或强有力定位语
- [ ] 每个项目有"关键决策"段落
- [ ] 所有可见文字有 `data-field` 属性
- [ ] GitHub链接直接写在 `href` 里
- [ ] 原型图槽用 `min-height` 不用固定 `height`
