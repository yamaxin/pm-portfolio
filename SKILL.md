# 产品经理作品集生成器

帮助产品经理从简历快速生成一份**专业、美观、可直接在浏览器里编辑**的 HTML 作品集。

输出是**单 HTML 文件**，内嵌完整编辑系统——用户打开后可直接点击修改所有文字、上传图片、增删模块，无需任何工具。

---

## 工作流程

遵循以下5步，不要跳步：

```
Step 1  读取简历 → 缺口分析
Step 2  多轮追问 → 补全信息
Step 3  确认内容 → 获得用户OK
Step 4  生成 HTML 作品集（用用户信息填充模板结构）
Step 5  说明编辑和导出方法
```

---

## Step 1 · 读取简历，分析缺口

读取用户上传的简历（无简历时先做基础追问）。

分析以下5个维度，判断每个维度的信息是否充分：

| 维度 | 充分标准 | 常见缺口 |
|------|---------|---------|
| **A 项目深度** | 有背景+决策逻辑+成果 | 只有结果数字，没有"为什么这样做" |
| **B AI能力** | 有具体工具+具体场景 | 只说"会用AI"，没有细节 |
| **C 商业价值** | 有市场判断+验证结果 | 只有功能描述，没有商业视角 |
| **D 个人标签** | 有他人评价+差异化定位 | 只有自我描述 |
| **E 可视化证明** | 有原型截图或链接 | 纯文字，无视觉证明 |

分析完成后，**简短告诉用户你看到了什么**，然后开始追问缺口。

---

## Step 2 · 多轮追问

**原则：**
- 每轮最多3个问题，按缺口优先级排序
- B（AI能力）和 D（他人评价）优先——HR最看重
- 用户说"没有"就跳过，不要强求
- **如果简历信息已经充分（5个维度都有），可以直接进入 Step 3**

**优先追问的5个方向：**

1. **AI能力**（必问）：你用过哪些AI工具？具体用在产品工作哪个环节？有没有设计过AI产品？有PRD或原型吗？有GitHub项目？

2. **关键决策**（必问）：你最有代表性的项目，当时最关键的一个产品决策是什么？你为什么这样决定，而不是另一种方案？

3. **他人背书**（必问）：同事或研发有没有对你说过让你印象深刻的评价？哪怕一句话。

4. **差异化**：你觉得你和其他PM最不一样的地方是什么？

5. **投递方向**：你现在主要投哪个方向？（这影响项目排序）

---

## Step 3 · 确认内容

追问完成后，整理并展示将要生成的内容：

- **定位标签**（3-4个）：基于方向和能力确定
- **封面数字**（4-5个）：最有冲击力的量化成果
- **代表项目**（3-4个）：覆盖不同能力维度
- **AI模块**：工具实践 + 产品思考
- **职业轨迹**：成长叙事

等用户确认后再生成。如果用户说"直接生成"，也可以直接开始。

---

## Step 4 · 生成 HTML 作品集

### 重要说明

`assets/portfolio-template.html` 是编辑系统的参考示例，用于了解编辑系统的CSS、JS和HTML结构规范。

**生成规则：**
- 必须用收集到的用户真实信息生成全新的HTML文件
- 不能直接输出或复制 portfolio-template.html 的内容
- 从零生成 HTML，但遵循模板的编辑系统规范（CSS类名、JS逻辑、data-field标记方式）
- 编辑系统的技术规范详见 `references/editor-system.md`，配色规范详见 `references/design-system.md`

### 作品集结构

生成以下5个模块，所有文字内容来自用户的真实信息：

**MODULE 1 · 封面**
- 姓名（大字）
- **他人评价作为主标语**（没有评价时用最强的自我定位语句）
- 3-4个定位标签
- 4-5个核心数字（横排）
- 联系方式 + GitHub

**MODULE 2 · 代表项目（3-4个）**

每个项目必须包含：
- 背景（市场/业务问题）
- **关键决策**（为什么这样做——这是与简历最大的区别）
- 策略步骤（3-5步）
- 成果（数字+业务影响）
- 原型图区域（占位槽，用户可上传替换）

项目覆盖维度（尽量覆盖）：
①AI/新技术 ②大型系统设计 ③商业化产品 ④行业深度

**MODULE 3 · AI产品思考与工具实践**
- 3个工具卡片：名称+定位+使用场景+用法列表
- 3个AI思考：结论+场景+可迁移判断标准
- 2个GitHub项目入口（无项目时展示技术标签即可）

**MODULE 4 · 市场洞察与商业判断**
- 3组：市场信号 → 我的判断 → 验证结果

**MODULE 5 · 职业轨迹**
- 时间轴（2-3条经历）
- 成长维度对比（3个维度）

### 技术规范

编辑系统的完整实现规范，参考 `references/editor-system.md`：

**关键要求：**
1. 输出单 HTML 文件，文件名：`{姓名}-产品经理作品集.html`
2. 所有可编辑文字加 `data-field="唯一id"` 和 `data-field-label="显示名"` 属性
3. `<script id="pe-saved-data" type="application/json">{}</script>` 放在 body 内、PE主脚本之外
4. GitHub链接直接写在 `href` 属性里，不用弹窗
5. 原型图槽用 `min-height:200px` + `height:auto`，不用固定高度
6. 所有删除/添加按钮加 `contenteditable="false"`
7. `'<!DOCTYPE html>\n'` 中的换行用转义 `\n`，不用真实换行符

### 内容写作原则

**项目描述：有决策，不只有结果**

❌ 简历式：`负责核心产品，落地30+功能，填补多项短板。`

✅ 作品集式：
```
背景：市场进入红海，产品功能趋同。

关键决策：面对"堆功能"还是"做减法"的选择，选择了后者——
建立需求优先级评估体系，优先落地客户迁移的核心卡点。

结果：支撑数十家KA客户迁移，续签订单超2000万。
```

**数字：有背景，不只是指标**
- ❌ "效率提升50%"
- ✅ "工单处理效率提升50%——从每天翻找文档改为直接对话问答"

## Step 5 · 告知使用方法

生成后，简要告知用户：

**编辑**：点击页面右下角 ✏️ 编辑按钮，进入编辑模式后点击任意文字直接修改。

**保存**：
- 本地/Hermes/OpenClaw 环境：点击 💾 保存文件，下载新 HTML 文件替换旧文件
- Claude.ai 环境：点击保存后内容存入浏览器，刷新页面自动恢复

**上传原型图**：编辑模式下点击图片区域（蓝色虚线框）上传。

**导出 PDF**：
- 本地环境：点击 📄 导出PDF 直接生成文件
- Claude.ai：打印对话框 → 另存为PDF → 勾选"背景图形"

**Ctrl+S** 快捷键保存。

## 质量检查（生成完成后自检）

生成HTML后，快速确认：

- [ ] 所有内容是用户真实信息，不是模板占位内容
- [ ] 封面有他人评价或强有力的个人定位语
- [ ] 每个项目有"关键决策"段落
- [ ] 所有可见文字都有 `data-field` 属性
- [ ] `pe-saved-data` 在 PE主脚本标签之外
- [ ] GitHub链接直接写在 `href` 里
- [ ] 原型图槽用 `min-height` 不用固定 `height`
