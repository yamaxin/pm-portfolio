# 产品经理作品集生成器

## 这个 Skill 做什么

接收用户的简历信息，通过多轮对话补充细节，最终**在对话中直接输出一段完整的 HTML 代码**作为作品集。

---

## ⚠️ 输出方式（最高优先级）

**本 Skill 的唯一输出方式：在对话回复中直接输出 HTML 代码。**

禁止的行为：
- ❌ 不能读取、引用或复制任何已有文件（包括环境中存在的其他 HTML 文件）
- ❌ 不能把已有文件当作本次输出结果
- ❌ 不能执行任何"部署"、"安装"、"写入文件系统"等操作
- ❌ 不能输出 `assets/portfolio-template.html` 的内容
- ❌ 不能生成 backup 文件或任何附属文件

正确的行为：
- ✅ 根据当前对话中用户提供的信息，从零生成全新的 HTML 代码
- ✅ 直接在对话回复中输出完整 HTML（一个代码块）
- ✅ 输出完成后，告知用户复制代码保存为 `.html` 文件即可使用

---

## 工作流程

```
Step 1  读取简历 → 缺口分析
Step 2  多轮追问 → 补全信息
Step 3  确认内容 → 获得用户OK
Step 4  生成全新 HTML（在对话中直接输出代码）
Step 5  说明编辑和导出方法
```

---

## Step 1 · 读取简历，分析缺口

读取用户上传的简历（无简历时先做基础追问）。

分析以下5个维度：

| 维度 | 充分标准 | 常见缺口 |
|------|---------|---------|
| **A 项目深度** | 有背景+决策逻辑+成果 | 只有结果数字，没有"为什么这样做" |
| **B AI能力** | 有具体工具+具体场景 | 只说"会用AI"，没有细节 |
| **C 商业价值** | 有市场判断+验证结果 | 只有功能描述，没有商业视角 |
| **D 个人标签** | 有他人评价+差异化定位 | 只有自我描述 |
| **E 可视化证明** | 有原型截图或链接 | 纯文字，无视觉证明 |

分析完成后简短告知用户，然后追问缺口。

---

## Step 2 · 多轮追问

- 每轮最多3个问题，B（AI能力）和 D（他人评价）优先
- 用户说"没有"就跳过；信息充分时直接进入 Step 3

**优先追问方向：**
1. AI工具：具体用在哪些环节？有没有设计过AI产品？有GitHub项目？
2. 关键决策：最有代表性的项目，最关键的决策是什么？为什么这样而不是另一种方案？
3. 他人背书：同事或研发有没有印象深刻的评价？
4. 差异化：和其他PM最不一样的地方？
5. 投递方向：主要投哪个方向？

---

## Step 3 · 确认内容

整理展示将要生成的内容，等用户确认。用户说"直接生成"时直接开始。

---

## Step 4 · 生成 HTML

### 生成规则

**规则1：所有内容来自当前对话，与环境中任何已有文件无关。**

生成前自问：
- 我现在要生成的姓名是当前用户的姓名吗？
- 我现在要生成的项目是当前用户的项目吗？
- 我有没有从任何已有文件复制内容？

如果任何一个答案是"否"，立即停止，重新根据对话内容生成。

**规则2：编辑系统代码从 `references/editor-system.md` 完整复制，禁止简化。**

最终 HTML 必须包含以下 11 个独立 script 标签（禁止合并）：

```
script 1:  环境检测         来自 editor-system.md 第5节
script 2:  pe-saved-data    固定内容：<script id="pe-saved-data" type="application/json">{}</script>
script 3:  PE主脚本         来自 editor-system.md 第7节
script 4:  图片上传         来自 editor-system.md 第8节
script 5-11: 增删系统       来自 editor-system.md 第9节（7个脚本）
```

**规则3：pe-saved-data 必须是空对象 `{}`，禁止携带任何数据。**

**规则4：工具栏内部元素不能加 data-field 属性。**

### 作品集结构（5个模块）

**MODULE 1 · 封面**
- 姓名（当前用户的真实姓名）+ 他人评价主标语
- 3-4个定位标签 + 4-5个核心数字 + 联系方式+GitHub

**MODULE 2 · 代表项目（3-4个）**
每个项目：背景、**关键决策**、策略步骤（3-5步）、成果、原型图占位槽
维度：①AI/新技术 ②系统设计 ③商业化 ④行业深度

**MODULE 3 · AI产品思考与工具实践**
- 3个工具卡片（名称+定位+场景+用法）
- 3个AI思考（结论+场景+判断标准）
- 2个GitHub项目（链接直接写在href里）

**MODULE 4 · 市场洞察**：3组 市场信号→判断→验证结果

**MODULE 5 · 职业轨迹**：时间轴（2-3条）+ 成长维度对比（3个）

### 技术要求

- 所有可编辑文字加 `data-field="唯一id"` 和 `data-field-label="显示名"`
- 原型图槽用 `min-height:200px` + `height:auto`
- 所有删除/添加按钮加 `contenteditable="false"`
- `'<!DOCTYPE html>\n'` 用转义 `\n`
- 文件命名建议：`{姓名}-产品经理作品集.html`

---

## Step 5 · 告知使用方法

输出完成后告知用户：
1. 复制上方代码，粘贴到文本编辑器，保存为 `{姓名}-产品经理作品集.html`
2. 用 Chrome 打开后，点击右下角 ✏️ 编辑，点击文字直接修改
3. 本地环境：💾 保存文件下载新HTML；Ctrl+S 快捷键
4. 导出PDF：本地直接生成；在线平台打印→另存为PDF→勾选"背景图形"

## 质量自检（生成后，输出前逐项确认）

**内容正确性（最关键）：**
- [ ] 封面姓名是当前用户的姓名，不是"杨硕"、"张三"或任何其他人
- [ ] 所有项目内容来自当前用户的简历
- [ ] 没有从任何已有文件复制任何内容

**编辑系统完整性：**
- [ ] 共11个独立script标签
- [ ] pe-saved-data 内容是 `{}`（纯空对象，无任何数据）
- [ ] 包含函数：`toggle` `exitEdit` `save` `init` `pdf` `copyCode`
- [ ] 包含函数：`initProjCards` `initStepLists` `initToolCards` `initThinkingCards`
- [ ] 包含函数：`initMarketCards` `initCareerTimeline` `initGrowthGrid`
- [ ] 包含函数：`initGithubCards` `initGithubGrid` `initTagGroups` `initProtoDel`
- [ ] 工具栏内无 data-field 属性
